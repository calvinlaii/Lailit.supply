## Spacing Scale

Declared values (4-point baseline; multiples of 4 only):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| 2xs | 4px | `p-1` / `gap-1` | Icon-to-text gap inside buttons, badge inner padding |
| xs | 8px | `p-2` / `gap-2` | Tight inline spacing; chip padding |
| sm | 12px | `p-3` / `gap-3` | Form field internal padding (vertical) |
| md | 16px | `p-4` / `gap-4` | Default body element spacing; nav link spacing |
| lg | 24px | `p-6` / `gap-6` | Card padding; section internal spacing |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between adjacent blocks |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks on mobile |
| 3xl | 64px | `p-16` / `gap-16` | Page-level vertical rhythm on desktop |
| 4xl | 96px | `p-24` / `gap-24` | Hero top/bottom padding on desktop (editorial breathing room — D-04) |
| 5xl | 128px | `p-32` / `gap-32` | Marketing-only "premium whitespace" between hero and pricing teaser on desktop ≥1024px |

**Exceptions:** Mobile nav hamburger button has a 44×44px hit area (icon-only touch target — Apple HIG / WCAG 2.5.5 minimum). The visual icon is 24px; padding fills to 44px. Documented as the **only** non-multiple-of-4 dimension and reserved for icon-only touch targets across all phases.

**Container rules:**
- Mobile (<640px): page horizontal padding = 16px (`px-4`)
- Tablet (640–1023px): page horizontal padding = 32px (`px-8`)
- Desktop (≥1024px): max content width = 1200px, centered (`max-w-[1200px] mx-auto px-12`)
- Long-form prose (legal pages): max width = 720px (`max-w-[720px]`) for 60–75ch optimal reading measure

## Typography

Declared sizes (4 sizes, 2 weights — strict 60/30/10 typographic hierarchy):

| Role | Size (mobile) | Size (desktop ≥1024px) | Weight | Line Height | Tracking |
|------|---------------|------------------------|--------|-------------|----------|
| Display (hero headline) | 48px / 3rem | 96px / 6rem | 600 (semibold) | 1.05 | -0.03em (tight) |
| Heading (section H2, pricing card price) | 28px / 1.75rem | 40px / 2.5rem | 600 (semibold) | 1.15 | -0.02em |
| Body (paragraphs, nav links, button labels) | 16px / 1rem | 16px / 1rem | 400 (regular) | 1.5 | 0 |
| Label (pricing feature bullets, footer, fine print, badges) | 14px / 0.875rem | 14px / 0.875rem | 400 (regular) | 1.45 | 0 |

**Weight rule:** ONLY 400 (regular) and 600 (semibold). No 500, no 700, no 800, no italics anywhere in marketing surface. This enforces the "spacious + editorial" aesthetic (D-04) — restraint over flourish.

**Font family rule:** All text uses `var(--font-geist-sans)`. The IDR price numerals (e.g. "Rp 449.000") on pricing cards use `var(--font-geist-sans)` with `font-variant-numeric: tabular-nums` so digits align in the side-by-side card layout (D-10).

**Body line-height rationale:** 1.5 on body / 1.45 on label / 1.15 on headings / 1.05 on display follows the standard "tighter as larger" rhythm and gives legal pages comfortable reading at 16px × 1.5 = 24px line height.

**Heading hierarchy mapping:**
- `<h1>` on `/` = Display (hero only — exactly one per page)
- `<h1>` on `/pricing`, `/login`, legal pages = Heading
- `<h2>` everywhere = Heading
- `<h3>` (pricing card plan name e.g. "Bulanan") = Body @ 600 weight
- All other text = Body @ 400 or Label @ 400

## Color

Pure monochrome palette — no accent (CTX D-02). Component previews (rendered inside the hero animated demo) provide the only on-page color. This is the brutalist/editorial commitment.

| Role | Value | Tailwind token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#FFFFFF` | `bg-white` | Page background (`<body>`), hero background, legal page background |
| Secondary (30%) | `#FAFAFA` | `bg-neutral-50` | Pricing card surface, login form card, footer background, alternating section bands |
| Accent (10%) | `#0A0A0A` | `bg-neutral-950` / `text-neutral-950` | Headlines, body text, primary CTA fill, nav logo, hero animated demo border |
| Border | `#E5E5E5` | `border-neutral-200` | Card borders, divider lines, input field borders, footer top border |
| Muted text | `#737373` | `text-neutral-500` | Fine print ("Batalkan kapan saja"), legal page metadata, footer copyright, instructional copy on `/login` |
| Destructive | `#DC2626` | `text-red-600` | Form validation error text on `/login` (error state copy below). **Never used as a fill, never decorative.** |

**Accent reserved for (explicit list, no expansion):**
- Page text (`#0A0A0A` is the foreground "ink")
- Primary CTA button fill (`Mulai Sekarang`, `Berlangganan Bulanan`, `Beli Lifetime`, `Kirim Magic Link`)
- Active/hover state on nav links (underline)
- Hero animated demo container 1px border
- Pricing card "Paling Populer" badge (filled black pill with white text)

**Forbidden in Phase 1:**
- Color gradients of any kind
- Drop shadows beyond the single elevation shadow defined below
- Any chromatic color (red is destructive-only, never decorative)
- Background patterns or textures

**Elevation:** ONE shadow token only. `shadow-[0_1px_2px_rgba(10,10,10,0.04),0_4px_12px_rgba(10,10,10,0.04)]` applied to: pricing cards (resting state), login form card. No other elevation. No hover-lift shadow change (interaction is signaled via border + cursor only).

**Radius:** `--radius: 12px` for cards; `--radius-sm: 8px` for buttons/inputs; `--radius-full: 9999px` for the "Paling Populer" pill badge.

## Hero Animation Spec

CTX D-05 commits to "one looping animated component demo playing live." The animation is the brand's first impression — it must feel premium, smooth, and performance-safe.

**What animates** (Phase 1 placeholder — real component swapped in Phase 3):

A single self-contained CSS-only "magnetic hover button" demo:
- A black pill-shaped button with the label `Hover me` (English here on purpose — a single English word is forgivable inside a demo about interactions, like a code sample is in English).
- Below it, a 1px black caret cursor that auto-orbits around the button on a 4-second loop, "magnetizing" toward the button as it approaches (transform-origin offset, eased), then pulling away.
- The button itself scales `1.0 → 1.04 → 1.0` and translates by `±4px` toward the cursor on each pass.
- A subtle 1px label below the demo: `Live demo — salin kode dalam satu klik`.

**Why CSS-only:**
- Zero JS bundle cost on first paint.
- No Mux dependency yet (that lands Phase 3).
- Survives `prefers-reduced-motion` gracefully (see below).
- Works on iOS Safari without autoplay quirks (deferred to Phase 5 spike).

**Implementation token contract:**

| Property | Value |
|----------|-------|
| Animation duration | 4s |
| Iteration | infinite |
| Easing | `cubic-bezier(0.4, 0, 0.2, 1)` (Tailwind's default `ease-in-out`) |
| Button scale | `1.0` → `1.04` → `1.0` |
| Cursor path | 4 keyframes orbiting at radius 80px from button center |
| Container | `aspect-ratio: 16/9`, max-w-[960px], 1px neutral-200 border, 12px radius, white background |
| Inner padding | 64px (gives the demo breathing room) |

**Reduced motion:**
- `@media (prefers-reduced-motion: reduce)`: the cursor + button motion stops entirely. The button rests in the resting state (scale 1, translate 0). The demo container still renders. Caption changes to `Demo statis — animasi dimatikan` (server-rendered, not JS-detected; uses CSS-only with a `<noscript>` fallback unnecessary because no JS is required).

**Performance budget:**
- `<HeroAnimatedDemo />` ships zero KB of JS to the client (use the `'use client'` boundary nowhere; pure CSS in a Server Component).
- No Lottie, no Three.js, no Framer Motion in Phase 1. (Motion library is reserved for Phase 5 hover effects per STK.)
- Total LCP impact target: hero must hit LCP <2.5s on 3G Fast.

**Phase-3 swap-in plan:** When Phase 3 ships real components, replace the CSS-only demo with a real `<MuxPlayer>` of the actual component preview, keeping the same container dimensions and 12px radius. The contract above defines the slot, not the eternal contents.

## Interaction States

Every interactive element declares 4 states. Focus visibility is mandatory (WCAG 2.4.7).

### Primary CTA button (`Mulai Sekarang`, `Berlangganan Bulanan`, `Beli Lifetime`, `Kirim Magic Link`)

| State | Visual |
|-------|--------|
| Resting | Background `neutral-950` (`#0A0A0A`), text white, padding `12px 24px`, radius 8px, weight 600, body size, no border |
| Hover | Background shifts to `neutral-800` (`#262626`). Transition: `background-color 150ms ease-out`. Cursor: pointer. **No scale change, no shadow change** (editorial restraint). |
| Focus-visible (keyboard) | Resting visual + 2px outline `outline-neutral-950` with 2px offset → reads as a black ring outside the button |
| Active (mouse down) | Background `neutral-900` (`#171717`); no transform |
| Disabled / Stubbed | Cursor: `not-allowed`; opacity 0.5; `aria-disabled="true"`. Used only for stubbed pricing CTAs in Phase 1 (CTX D-12). Hover does nothing. Does not navigate. |
