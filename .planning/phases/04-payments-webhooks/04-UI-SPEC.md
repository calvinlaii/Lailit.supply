## Color

Tailwind v4 `@theme` inline — no tailwind.config.js. All colors are OKLCH CSS variables.

| Role | CSS Variable | Resolved (approx) | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background: oklch(1 0 0)` | #ffffff | Page backgrounds, all surfaces |
| Secondary (30%) | `--muted: oklch(0.97 0 0)` | ~#f7f7f7 | Card footers, hover states, muted sections |
| Accent (10%) | `--primary: oklch(0.205 0 0)` | ~#1a1a1a / neutral-950 | See reserved-for list below |
| Destructive | `--destructive: oklch(0.577 0.245 27.325)` | ~red-600 | No destructive actions in Phase 4 UI |

Accent (neutral-950 / primary) reserved for:
- Pricing CTA buttons ("Berlangganan Bulanan", "Beli Lifetime") — bg-neutral-950 text-white
- "Masuk ke Dashboard" button in welcome email — consistent primary CTA treatment
- Avatar fallback background — bg-neutral-950
- Active/selected nav state
- Focus-visible outline — outline-neutral-950
- "Kelola Langganan" account page button (primary CTA)
- Status badge: `membership_status = 'active'` → bg-neutral-950 text-white (solid badge)

Status badge color mapping for `/account`:
- `active` ("Aktif") → Badge variant `default` (bg-neutral-950 text-white)
- `canceled` ("Dibatalkan") → Badge variant `outline` (border-border text-foreground)
- `expired` ("Kedaluwarsa") → Badge variant `destructive` (bg-destructive/10 text-destructive)

Source: `src/app/globals.css`, `src/components/ui/badge.tsx`, `src/components/marketing/pricing-card.tsx`

## Layout Contract

### `/account` Page

- Inherits `(dashboard)` layout: sticky DashboardNav (h-16, bg-white, border-b) + `<main id="main">` wrapper
- Page max-width: `max-w-[1200px] mx-auto` — consistent with all dashboard pages
- Horizontal padding: `px-4 sm:px-8 lg:px-12` — consistent with dashboard page pattern
- Vertical padding: `py-16` top — consistent with dashboard page
- Layout: single column, centered card, max-width `max-w-[480px]` for membership card to avoid oversized form feel
- DashboardNav gets a new "Akun" link — placed before UserMenu in the nav bar (left of user avatar)

### Account Membership Card anatomy

```
[Card]
  [CardHeader]
    Plan name (h2, text-base font-semibold)   [Status Badge]
  [CardContent]
    Expiry date line (text-sm text-neutral-500)
    Canceled grace period note (text-sm text-neutral-500) — only when status=canceled
  [CardFooter]
    [Button full-width] "Kelola Langganan" → external Mayar Customer Portal (target="_blank")
```

### Pricing Page CTA update (existing component, minimal change)

- `href` on both CTA `<a>` elements changes from `"#"` to `process.env.NEXT_PUBLIC_MAYAR_MONTHLY_URL` / `NEXT_PUBLIC_MAYAR_LIFETIME_URL`
- Remove `aria-disabled="true"`, `data-stub="true"`, `tabIndex={-1}`, `cursor-not-allowed`, `opacity-50`, `pointer-events-none`
- Same visual styling retained: `bg-neutral-950 text-white px-6 py-3 rounded-lg text-base font-semibold`
- Keep `target="_self"` (same tab per D-02)

## Interaction Contract

### Pricing CTA → Mayar Checkout

- User clicks "Berlangganan Bulanan" or "Beli Lifetime"
- Full-page navigation to Mayar hosted link (same tab, no `target="_blank"`)
- No loading spinner needed — standard link navigation
- Back button returns user to `/pricing`

### Webhook Handler (no user-facing UI — backend only)

- `POST /api/webhooks/mayar?token=...` — no UI interaction

### Account Page — "Kelola Langganan"

- `<a href={MAYAR_CUSTOMER_PORTAL_URL} target="_blank" rel="noopener noreferrer">`
- Opens Mayar Customer Portal in new tab
- Rendered as Button (variant `outline` or `default` — use `default` for primary weight)
- No confirmation dialog required — navigates to external site, not a destructive action

### DashboardNav — "Akun" link

- `<Link href="/account">` — standard Next.js Link
- Active state: `font-semibold text-neutral-950` when on `/account`
- Inactive state: `text-neutral-500 hover:text-neutral-950` transition-colors 150ms

### Status Badge rendering rules

| `membership_status` | `lifetime_purchased` | Badge variant | Label |
|---------------------|---------------------|---------------|-------|
| `'active'` | false | `default` (dark) | Aktif |
| `'active'` | true | `default` (dark) | Aktif — Seumur Hidup |
| `'canceled'` | — | `outline` | Dibatalkan |
| `'expired'` | — | `destructive` | Kedaluwarsa |
| null (no membership) | — | `outline` | Belum berlangganan |

## Accessibility Contract

- All interactive elements carry ARIA labels in Bahasa Indonesia (consistent with existing `aria-label="Buka menu akun"`, `aria-label="Keluar dari akun"`)
- "Kelola Langganan" link: `aria-label="Kelola langganan di Mayar — tautan eksternal"` (communicates new-tab behavior)
- Status badge: rendered as `<span>` (not button) — informational only, no interaction
- Account page h1: `Akun` (single h1, consistent with dashboard page pattern of one h1 per page)
- Focus ring: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-neutral-950` — inherited site-wide pattern
- Reduced motion: existing `@media (prefers-reduced-motion: reduce)` in globals.css covers all transitions
